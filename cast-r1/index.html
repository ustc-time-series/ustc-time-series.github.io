项目简介

Cast-R1 是一个工具增强型自主交互时间序列预测智能体， 旨在推动时间序列预测从传统的单次模型推断， 走向基于工具调用、情境推理和多轮交互的智能体式预测范式。 与 CastMind（星思）通过独立 Feature Encoder 将时序编码为 Embedding 再送入 LLM 的 基础模型路线不同，Cast-R1 直接将原始时间序列作为 Token 输入语言模型， 不依赖独立编码器，而是将数值序列、统计特征、Meta Data、外生变量、 领域描述和任务情境等信息统一 Token 化，使模型能够端到端地感知任务背景、 数据语义和预测目标。

Cast-R1 的训练路线不是为了重新训练一个时序基础模型底座， 而是围绕已有 LLM 的智能体能力逐级增强： Cast-R1-SFT 让模型会做，掌握预测任务格式、Token 输入理解、工具调用协议和标准工作流； Cast-R1-Reasoning 让模型会想，理解上下文、工具反馈和预测不确定性； Cast-R1-Agentic RL 让模型会行动，在 Harness 工具环境中学习何时调用工具、 调用哪个工具、如何组合工具链、如何更新上下文，以及何时停止交互并输出预测。

三大核心能力

Cast-R1 的三阶段训练对应三类能力递进：SFT 让模型会做，Reasoning 让模型会想，Agentic RL 让模型会行动。

🛠️

Cast-R1-SFT：会做

Task Format · Tool Protocol · Standard Workflow

通过高质量专家轨迹监督学习，让模型掌握基本预测任务格式、 时间序列 Token 输入理解、工具调用协议和标准工作流。 模型学习如何读取原始时序 Token、理解情境上下文、调用数据预处理 / 特征抽取 / 小模型工具，并整合结果输出预测。

🧩

Cast-R1-Reasoning：会想

Context Reasoning · Tool-feedback Reasoning

通过推理数据蒸馏、CoT 数据构造、过程监督或偏好优化， 强化模型的上下文推理和工具反馈推理能力。 目标不是让模型只会调用工具，而是理解工具返回结果的含义， 能分析趋势、周期、异常、外部事件影响和预测不确定性。

🔄

Cast-R1-Agentic RL：会行动

Tool Action Policy · Context Update · Stop Decision

在 Harness 工具环境中进行多轮交互，重点优化行动策略： 何时调用工具、调用哪个工具、如何组合工具链、如何利用工具结果更新上下文、 何时停止交互并输出预测。奖励来自最终预测误差、过程质量、工具调用成本、 工具结果有效性和上下文更新质量。

Harness 工具环境

Cast-R1 借助 Harness Engineering 思想构建了面向时间序列预测的工具环境， 为智能体提供可操作、可反馈、可验证的外部行动空间， 使模型能够在预测过程中主动感知数据、分析模式、调用模型、修正判断：

数据预处理工具

归一化、去噪、重采样、异常点标记，为后续分析和模型调用提供干净规范的时序输入。

缺失值处理工具

插值补全、缺失片段标记与不确定性估计，支持非完整时序数据下的鲁棒预测。

统计诊断工具

平稳性检验、自相关与偏自相关分析、趋势检验，帮助模型理解序列的基本统计属性。

特征抽取工具

趋势分解（STL / EMD）、周期强度提取、统计特征计算，将数值序列转化为可推理的语义特征。

频域分析工具

傅里叶变换、频谱分析、主频提取，识别隐藏的周期结构和频率域模式，辅助趋势归因。

外生变量处理工具

协变量时间对齐、外部信号融合与相关性分析，将多源异构情境信号引入预测推理过程。

预训练小模型调用

DLinear、PatchTST 等轻量化专家模型按需调用，提供快速数值参考，辅助智能体形成预测判断。

预测结果评估工具

误差评估、置信区间验证与多模型结果对比，为智能体的反思修正提供可验证的外部反馈信号。

  input_tokens ← 原始时间序列 + 情境上下文 Token

  standard_workflow ← SFT 学习的基础预测流程与工具协议

Cast-R1-SFT → Cast-R1-Reasoning → Cast-R1-Agentic RL

Cast-R1 采用三阶段递进训练路径：SFT 让模型会做，Reasoning 让模型会想， Agentic RL 让模型会行动。这一路线面向已有语言模型， 不以构建新的时序基础模型底座为核心目标，而是训练模型在 Harness 工具环境中进行 自主工具调用、上下文推理和多轮预测决策。

Stage 1 · Cast-R1-SFT

监督微调
任务格式与标准工作流

输入：高质量专家轨迹 + 工具调用协议 + 标准预测流程

负责让模型学会基本预测任务格式、时间序列 Token 输入理解、工具调用协议和标准工作流。 专家轨迹覆盖如何读取原始时序 Token、理解情境上下文、调用数据预处理 / 特征抽取 / 小模型工具，以及如何整合结果输出预测。

能力：会做 · 任务格式 · 工具协议 · 标准流程
Stage 2 · Cast-R1-Reasoning

推理增强
上下文与工具反馈理解

输入：推理数据蒸馏 + CoT 构造 + 过程监督 / 偏好优化

负责强化模型的上下文推理和工具反馈推理能力。这个阶段不一定需要 RL， 可以先通过推理数据蒸馏、CoT 数据构造、过程监督或偏好优化， 让模型学会分析趋势、周期、异常、外部事件影响，并解释工具结果和预测不确定性。

能力：会想 · 上下文推理 · 工具反馈推理 · 不确定性解释
Stage 3 · Cast-R1-Agentic RL

智能体强化学习
行动策略优化

输入：Harness 工具环境交互 + 多源奖励信号

最核心阶段。模型在 Harness 构建的工具环境中进行多轮交互，学习何时调用工具、 调用哪个工具、如何组合工具链、如何利用工具结果更新上下文、 何时停止交互并输出预测。重点优化行动策略，而不是基础表征能力。 奖励可来自最终预测误差、过程奖励、工具调用成本、工具结果有效性和上下文更新质量。

能力：会行动 · 工具选择 · 工具链组合 · 上下文更新 · 停止决策
