# 科学时序数据建模子页面设计

## 目标

在网站“研究方向”体系中建设“科学时序数据建模”子页面，使访客能够快速理解科学时序数据的定义边界、动力系统差异、跨尺度建模问题、核心任务与迁移评测方法。页面以 Time Series for Science Benchmark 规划作为研究方向的具体落地框架，但不把尚未冻结的数据清单或规划内容表述为已经完成的成果。

## 内容依据

本轮增强以本地规划文档 `时序数据分析驱动的学科规划.md` v0.3（2026-07-20）为主要内容来源。页面提炼其中稳定的研究框架：

- 科学时序数据必须对应真实的时间、循环或实验过程语义；
- 研究对象由科学系统、物理描述尺度和适配预算三个正交维度组织；
- 真实观测与真实实验数据构成主赛道，计算模拟构成辅助赛道；
- 核心评测是零样本、少样本和全量微调形成的迁移曲线；
- 四类科学系统下设置八个互补任务包，用最小覆盖集刻画不同动力学与观测范式；
- 除任务误差外，还需要评估物理一致性、不确定性、极端事件、数据效率与负迁移。

## 方案比较

### 方案一：仅增加导航下拉入口

创建详情页并在“研究方向”下拉菜单中增加入口。改动最小，但主页正文缺少对应研究卡片，方向可见性偏弱。

### 方案二：仅增加主页研究卡片

在主页“研究方向”区域增加介绍卡片，通过卡片进入详情页。正文可见性较好，但顶部导航与其他研究子页不一致。

### 方案三：主页卡片、导航入口与独立详情页同步增加（采用）

在主页增加完整研究方向卡片，在所有现有“研究方向”下拉菜单中增加入口，并创建独立详情页。该方案与当前预测、分类与异常检测页面的访问路径一致，可发现性和结构一致性最好。

## 信息架构

### 主页入口

- 在根主页“研究方向”项目网格中增加一张全宽卡片。
- 卡片标题为“科学时序数据建模”。
- 简介强调面向复杂科学系统，建模多尺度、强耦合、非平稳、稀疏或不规则观测及机理约束下的动态演化。
- 卡片用四条元信息概括：科学数据特点、核心任务、技术范式、应用领域。
- 主按钮进入 `scientific-time-series/`。

### 顶部导航

- 在所有采用现有统一顶部导航的页面中，将“科学时序数据建模”加入“研究方向”下拉菜单。
- 新详情页将该菜单项标记为当前页面。
- 保持现有研究子页顺序：时间序列预测、时间序列分类与异常检测、科学时序数据建模。

### 独立详情页

- 路径：`scientific-time-series/index.html`。
- 复用现有研究详情页的导航、青绿色视觉、卡片圆角、响应式断点和页脚风格。
- 页面标题与描述使用中英文研究术语，但正文以中文为主。

## 页面内容

### 首屏与研究定位

- 主标题：科学时序数据建模。
- 英文副标题：Scientific Time Series Modeling。
- 导语强调从复杂科学观测中学习跨系统、跨尺度的动态规律，服务预测、重建、适配与科学发现。
- 三个概览标签调整为：跨系统动力学、跨尺度科学建模、零样本到全量适配。
- 增加一条研究定位语句：以科学动力系统和物理描述尺度的多样性为组织原则，评测科学时序基础模型的跨系统、跨尺度表征与适配能力。
- 页面明确标注 Benchmark 仍属于研究规划与评测框架，避免误导为已冻结发布的数据产品。

### 第一部分：什么是科学时序数据

- 保留多尺度、强耦合、非平稳、稀疏与机理约束等核心特征。
- 增加“过程语义”边界：时间轴必须对应物理、生物、化学、材料状态或实验循环的真实演化。
- 增加“观测过程”概念：科学时序是目标状态经过仪器、传感器或观测算子形成的记录。
- 用正反两列说明：多变量传感器、波形、空间场、时序光谱、控制轨迹属于典型科学时序；静态谱图、静态性质表、任意排序样本和仅有最终标签的数据不能仅凭一维横轴被视为时序。

### 第二部分：三维研究与评测框架

用一个清晰的“三维立方体”视觉结构呈现：

1. 科学系统维度：物理与宇宙、地球、生命、分子与材料；
2. 物理描述尺度：L0 电子/量子、L1 原子/分子、L2 介观/动理学、L3 连续介质/场、L4 系统/工程动力学；
3. 适配预算：零样本、少样本、全量微调。

附加两条边界：真实观测与真实实验为主赛道；计算模拟为辅助赛道，模拟到真实迁移单独评测。

### 第三部分：四类科学系统与八个任务包

按科学动力系统而不是行政学科名称组织卡片：

- 物理与宇宙系统：受控等离子体与核聚变、时域天文观测；
- 地球系统：大气—环境耦合、水文与水循环、地震与固体地球事件；
- 生命系统：生理—神经调节；
- 分子与材料系统：化学反应与分子动力、材料演化与服役退化。

每个系统卡片突出互补的动力学范式和迁移边界，不展开未经数据许可审计的下载入口。

### 第四部分：统一任务体系

用紧凑任务网格呈现七类任务：

1. 动力学预测；
2. 状态重建与逆问题；
3. 事件与变化检测；
4. 缺失与传感器鲁棒性；
5. 退化与剩余寿命；
6. 表征学习与跨领域适配；
7. 跨尺度桥接与闭合学习。

### 第五部分：迁移与科学有效性评测

- 用“零样本 → 1% → 5% → 10% → 全量微调”呈现适配曲线；少样本预算按独立科学实体而不是重叠窗口计算。
- 展示三类结果：任务性能、科学有效性、迁移与效率。
- 突出负迁移、最差领域表现、物理一致性、不确定性校准、极端事件与数据效率。
- 区分同尺度跨系统、同系统跨尺度、跨系统且跨尺度三种迁移方向，不合并成单一总分。

### 页面操作

- 主操作：返回首页研究方向。
- 次操作：查看论文列表。
- 不添加虚构或未经确认的论文、数据集和项目链接。

## 组件与依赖

- 页面为独立静态 HTML，不引入构建工具或外部 JavaScript 依赖。
- 导航下拉、页头刷新交互和移动端布局复用现有页面脚本与样式模式。
- 图形表达使用 CSS、文本和符号完成，不新增图片资源。
- 所有站内链接使用相对路径，适配 GitHub Pages 子目录部署。

## 错误与降级处理

- 禁用 JavaScript 时，页面正文与普通链接仍可正常阅读和访问；仅下拉菜单交互可能受限。
- 在窄屏设备上，多列卡片降级为单列，避免横向滚动。
- 所有链接采用真实存在的站内路径，测试中检查目标文件存在。
- 页面不依赖远程字体、接口或第三方资源，避免网络失败导致主体内容缺失。

## 验证标准

- 新详情页包含定义边界、三维框架、八个任务包、七类任务和迁移评测五个核心信息部分。
- 根主页研究方向区域包含新卡片和有效入口。
- 所有现有统一导航中的“研究方向”下拉菜单包含新入口。
- 新页面当前菜单状态正确，站内导航路径有效。
- 桌面与移动宽度下无明显溢出，标题、卡片和按钮层级清晰。
- Node 测试覆盖页面存在、三维框架、L0–L4、八个任务包、适配曲线、主页入口和导航同步。
- `git diff --check` 无空白错误；浏览器控制台无页面错误。

## 非目标

- 本次不新增论文、数据集或项目成果条目。
- 本次不把规划中的 Benchmark 描述为已经正式发布或已冻结的数据产品。
- 本次不罗列规划文档的全部数据锚点、许可细节和指标公式。
- 本次不重构全站共享样式或导航模板。
- 本次不部署或推送远程仓库，除非用户另行要求。
