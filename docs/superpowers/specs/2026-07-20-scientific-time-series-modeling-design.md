# 科学时序数据建模子页面设计

## 目标

在网站“研究方向”体系中新增“科学时序数据建模”子页面，使访客能够快速理解科学时序数据的独特挑战、研究问题、技术路线与典型应用。页面服务于研究方向展示，不承担论文目录或项目数据库功能。

## 方案比较

### 方案一：仅增加导航下拉入口

创建详情页并在“研究方向”下拉菜单中增加入口。改动最小，但主页正文缺少对应研究卡片，方向可见性偏弱。

### 方案二：仅增加主页研究卡片

在主页“研究方向”区域增加介绍卡片，通过卡片进入详情页。正文可见性较好，但顶部导航与其他研究子页不一致。

### 方案三：主页卡片、导航入口与独立详情页同步增加（采用）

在主页增加完整研究方向卡片，在所有现有“研究方向”下拉菜单中增加入口，并创建独立详情页。该方案与当前预测、分类与异常检测页面的访问路径一致，可发现性和结构一致性最好。

## 信息架构

### 主页入口

- 在根主页“研究方向”项目网格中增加一张全宽卡片。
- 卡片标题为“科学时序数据建模”。
- 简介强调面向复杂科学系统，建模多尺度、强耦合、非平稳、稀疏或不规则观测及机理约束下的动态演化。
- 卡片用四条元信息概括：科学数据特点、核心任务、技术范式、应用领域。
- 主按钮进入 `scientific-time-series/`。

### 顶部导航

- 在所有采用现有统一顶部导航的页面中，将“科学时序数据建模”加入“研究方向”下拉菜单。
- 新详情页将该菜单项标记为当前页面。
- 保持现有研究子页顺序：时间序列预测、时间序列分类与异常检测、科学时序数据建模。

### 独立详情页

- 路径：`scientific-time-series/index.html`。
- 复用现有研究详情页的导航、青绿色视觉、卡片圆角、响应式断点和页脚风格。
- 页面标题与描述使用中英文研究术语，但正文以中文为主。

## 页面内容

### 首屏

- 主标题：科学时序数据建模。
- 英文副标题：Scientific Time Series Modeling。
- 导语：面向自然、工程与生命系统中的动态观测，融合数据驱动学习、结构归纳偏置与科学机理，形成可预测、可解释、可迁移的时序模型。
- 三个概览标签：复杂科学观测、机理与数据融合、预测反演与发现。

### 第一部分：科学时序数据的特点

用四张并列卡片介绍：

1. 多尺度与非平稳：动态过程跨越不同时间尺度，并随环境或阶段发生分布变化。
2. 多变量与强耦合：多个物理量、生物量或系统状态通过复杂依赖共同演化。
3. 稀疏与不规则观测：采样频率、传感器覆盖和观测质量受实验或设备条件限制。
4. 机理与约束并存：守恒律、边界条件、因果结构和领域规律限制可能的演化路径。

### 第二部分：核心科学问题

采用四阶段研究链路：

1. 表征：统一编码异构变量、不同采样粒度和多源观测。
2. 预测：刻画长期演化、突变事件与不确定性。
3. 反演：从观测序列估计隐状态、未知参数与驱动因素。
4. 发现：识别稳定规律、关键机制和可验证的科学假设。

### 第三部分：技术路线

用三列卡片组织：

1. 数据驱动时序学习：自监督表征、时空依赖建模、多任务学习与基础模型。
2. 科学机理融合：物理约束、结构先验、微分方程和模拟数据协同。
3. 可信与可迁移建模：不确定性量化、分布外泛化、跨系统迁移与可解释分析。

底部用流程条概括：科学观测 → 多源时序表征 → 动态规律建模 → 可信预测与反演 → 科学发现与决策。

### 第四部分：应用领域

展示四个代表领域：气象与地球系统、能源与工程系统、生命与健康科学、材料与实验科学。每个领域说明典型数据与建模价值，不列未经确认的课题组成果。

### 页面操作

- 主操作：返回首页研究方向。
- 次操作：查看论文列表。
- 不添加虚构或未经确认的外部论文、数据集和项目链接。

## 组件与依赖

- 页面为独立静态 HTML，不引入构建工具或外部 JavaScript 依赖。
- 导航下拉、页头刷新交互和移动端布局复用现有页面脚本与样式模式。
- 图形表达使用 CSS、文本和符号完成，不新增图片资源。
- 所有站内链接使用相对路径，适配 GitHub Pages 子目录部署。

## 错误与降级处理

- 禁用 JavaScript 时，页面正文与普通链接仍可正常阅读和访问；仅下拉菜单交互可能受限。
- 在窄屏设备上，多列卡片降级为单列，避免横向滚动。
- 所有链接采用真实存在的站内路径，测试中检查目标文件存在。
- 页面不依赖远程字体、接口或第三方资源，避免网络失败导致主体内容缺失。

## 验证标准

- 新详情页包含四个信息部分及完整研究链路。
- 根主页研究方向区域包含新卡片和有效入口。
- 所有现有统一导航中的“研究方向”下拉菜单包含新入口。
- 新页面当前菜单状态正确，站内导航路径有效。
- 桌面与移动宽度下无明显溢出，标题、卡片和按钮层级清晰。
- Node 测试覆盖页面存在、内容结构、主页入口和导航同步。
- `git diff --check` 无空白错误；浏览器控制台无页面错误。

- 本次不重构全站共享样式或导航模板。
- 本次不部署或推送远程仓库，除非用户另行要求。
