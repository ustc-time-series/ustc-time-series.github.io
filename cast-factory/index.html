一句话定位： CastFactory（铸星坊）是一个 recipe-centric 的 LLM-based 时间序列预测微调训练框架—— 用一份 YAML recipe 描述完整实验，将 CPT、SFT 与 RLVR 三阶段训练、 数据构建、时序表示、可验证奖励、Agentic 工具调用与评测流程统一封装， 支撑 Time-R1 风格分阶段训练与 Cast-R1 风格工具增强决策两条研究路线。

项目简介

近年来，越来越多的研究尝试将大语言模型用于时间序列预测。然而，现有工作往往依赖 临时性的数据序列化方式、手工构造的 prompt 模板和分散的微调脚本， 导致模型训练难以复现、实验结果难以比较、方法扩展成本高。

CastFactory（铸星坊） 聚焦于 LLM-based TSF 模型微调训练这一核心问题， 将整个训练流程从"临时写脚本"升级为"数据格式统一、训练阶段清晰、评测协议标准、实验过程可复现"的 模型工厂式框架。

核心痛点

将 LLM 用于时间序列预测时，当前研究面临五类系统性挑战：

🔤

序列化缺乏统一标准

不同研究采用不同数值序列化方式——逗号分隔、自然语言、patch、时间戳拼接各不相同，细节设计差异直接影响 LLM 训练效果。

🧠

LLM 时序适应能力不足

通用 LLM 在自然语言语料上预训练，并不天然熟悉连续数值序列、趋势变化和周期模式，直接 SFT 难以形成稳定的时序表达能力。

🎯

SFT 目标与预测指标错位

SFT 训练目标是 token-level cross entropy，而预测真正关心的是 MAE、MSE、趋势方向准确率——学会生成格式不等于预测优化。

📄

生成式预测格式不稳定

LLM 输出文本需经 parser 转换为数值数组。输出长度错误、格式失效、数值无法解析都会影响评测，但多数框架缺乏生成有效性指标。

🔁

实验 recipe 缺乏可复现管理

base model、数据集、序列化方式、LoRA 配置、解码策略……任一设置变化都影响结果，但目前缺乏将这些设置沉淀为标准 recipe 的机制。

核心设计思想

CastFactory（铸星坊） 的设计思想可以概括为五个关键词： Standardize · Adapt · Instruct · Optimize · Reproduce。

  • S
    Standardize — 统一标准 统一数据接入（CSV reader + timestamp/ratio split + rolling windows + leakage 检查）、时序表示（statistics、textual_summary、context、markdown_table、discrete_token、numerical_patch、hybrid 七种内置 representation）、instruction 模板与评测协议，让不同数据集、不同模型、不同训练阶段在统一接口下运行。
  • A
    Adapt — 时序适应 通过 CPT 阶段让 LLM 接触序列化时序值与元信息（channel、domain、unit），在进入 SFT 之前先形成基础的时序表达能力——即 Temporal-adapted LLM。CPTDataset + TransformersCPTBackend 提供完整的 next-token 训练管线。
  • I
    Instruct — 指令预测 通过 SFT 阶段让模型学会按照预测任务指令输出未来序列。SFTDataset 将每条样本格式化为指令 + 可见时序上下文 + JSON 预测目标，Transformers backend 用 -100 mask prompt token，loss 仅作用在 response token 上。支持外部 .txt instruction 模板与丰富的占位符（cutoff_time、prediction_length、attr_meaning、target_channels…）。
  • O
    Optimize — 可验证奖励优化 TSF 天然具有 ground truth future values，因此可以将预测误差、格式有效性、推理结构、季节性/趋势/突变点结构转化为可验证奖励，通过 RLVR 阶段使优化目标更接近真实预测指标。RLVR 同时支持 single-turn rollout 与 verl 原生 AgentLoop 工具调用两种 workflow。
  • R
    Reproduce — 完整复现 每一次实验都沉淀为一份 YAML recipe + 一组 trace artifacts（recipe 快照、metrics、predictions、leaderboard、report、stage metadata、prompt/response/parsed rows/errors）。Experiment.fit() 与 Experiment.evaluate() 自动写出全部产物，支撑论文实验和开源生态建设。

CPT-SFT-RLVR 三阶段训练

CastFactory（铸星坊）将 LLM-based TSF 微调流程系统化为三个递进阶段， 对应 experiment.stage 的 cpt / sft / rlvr 三个一级取值， 由 Experiment.fit() 统一调度，每个阶段都有清晰的输入、目标与产出：

Stage 1 · CPT

持续预训练
时序适应阶段

输入：序列化时序值 + channel/domain/unit 元信息

CPTDataset 将预测窗口序列化为因果 LM 文本流，TransformersCPTBackend 完成 tokenize 并以 input tokens 同步作为 labels 训练 next-token 目标，支持 FSDP 多卡。

输出：Temporal-adapted LLM
Stage 2 · SFT

监督微调
指令预测阶段

输入：instruction + 可见时序上下文 + future-known 协变量

SFTDataset 将每条样本格式化为指令 + 时序表示 + JSON 目标输出，prompt token 用 -100 屏蔽，loss 仅作用于 response。支持外部 .txt 模板与 cutoff_time、prediction_length、target_channels、data_lookback 等占位符。

输出：Instruction-following TSF LLM
Stage 3 · RLVR

可验证奖励强化学习
奖励优化阶段

输入：滚动窗口 + ground truth future values + 奖励配置

RLVR 自动构造 rollout 行、verl 0.7.1 GRPO+vLLM 配置、reward 入口与启动命令；支持 single-turn 与 time_series_agent 两种 workflow，后者对接 verl 原生 AgentLoop 形成 Cast-R1 风格工具调用循环。

输出：Reward-optimized TSF LLM

RLVR 奖励设计

CastFactory 内置一组面向 TSF 的可验证奖励函数，可在 training.rewards 中按需组合； single-turn workflow 强调 结构化输出 + 数值精度，agentic workflow 进一步引入与时序结构相关的辅助奖励：

奖励项含义适用 Workflow
format检查 <think>…</think><answer>…</answer> 结构与可解析性，作为硬约束/前置门single-turn / agentic
accuracy基于 MAE / MSE 等点预测指标的准确性奖励single-turn
mse带温度的 MSE 有界奖励，作为主要数值精度信号single-turn
calibration基于分位数（q10/q90）的不确定性校准奖励single-turn
reasoning对推理段长度 / 与 horizon 的匹配性的辅助奖励single-turn
length预测长度是否与 horizon 一致agentic
normalized_mse归一化 MSE，跨窗口尺度更稳定agentic
change_point突变点位置一致性，鼓励抓住趋势翻转agentic
season_trend季节性 + 趋势分量的结构相似性agentic

所有奖励通过统一入口 castfactory.training.verl_reward_adapter:compute_score 暴露给 verl 0.7.1， 框架自动写出 rollout 数据集、reward 入口与启动命令，使外部 verl 运行时可直接执行 GRPO + vLLM 训练。

框架模块设计

CastFactory（铸星坊）由若干职责清晰、可独立扩展的子包构成，与代码仓库一一对应：

core · Recipe & Experiment

RecipeConfig 校验/规整 YAML，Experiment 装配数据、表示、模型、训练、奖励、评测、trace 组件，registry 管理可插拔的 backbone/trainer 注册表。

data · 数据流水线

CSV reader、timestamp/ratio split、rolling windows、leakage 检查、train-only 归一化工具，并将原始数据封装为可见性分离的 ForecastSample（observed / future-known / future-unknown 三段）。

representation · 时序表示

七种内置表示：statistics、textual_summary、context、markdown_table、discrete_token、numerical_patch、hybrid，统一输出 ModelInput 文本提示。

parsers · 输出解析

JSON、array、timestamp/value、<think>/<answer> 四类 forecast parser，附带 fallback 行为与解析上下文 ParseContext，将 LLM 文本输出稳定还原为数值数组。

models · 模型骨架

HuggingFace Causal LM backbone、TextConcatBridge、TextGenerationHead、PEFT/LoRA 适配器、vLLM 引擎适配器，构成与训练后端解耦的"积木式"模型层。

training · 三阶段训练

CPT/SFT/RLVR 各自的 Dataset + Trainer，含 TransformersCPTBackend、TransformersSFTBackend（FSDP）与 VerlBackend；prompt_template 支持外部 .txt 模板与丰富占位符。

training.agentic · Agent-Loop

Cast-R1 风格工具集：basic statistics、within-channel dynamics、forecast residuals、data quality、event summary 共 5 个分析工具，外加 predict_time_series（chronos2 / arima / patchtst / itransformer + ARIMA / last-value 本地兜底）。

rewards · 可验证奖励

FormatReward、AccuracyReward、CalibrationReward、MSEReward、ReasoningReward 与 agentic 路径下的 length、normalized_mse、change_point、season_trend，可在 recipe 中按需组合。

evaluation · 评测协议

三类评测器：StandardEvaluator、RollingEvaluator、ZeroShotEvaluator，配合点预测指标库；支持单协议或多协议同时评估并写入 leaderboard。

trace · 实验产物

RunStore 自动写出 recipe 快照、metrics、predictions、leaderboard、report 与 stage metadata，并提供 prompt/response/parsed rows/error 等扩展槽位，让"实验即可复现"。

cli · 统一命令入口

python -m castfactory.cli.run <recipe>.yaml --mode fit|evaluate 单命令驱动整个实验，支持 dotlist 覆盖（如 training.args.learning_rate=2e-4）。

Agentic RLVR：Cast-R1 风格工具调用

除了经典的 single-turn rollout，CastFactory 在 RLVR 阶段同时支持 time_series_agent workflow， 原生对接 verl 0.7.1 的 AgentLoop，让模型通过多轮工具调用形成"分析—预测—验证—回答"的闭环决策过程。

两种 Rollout Workflow

📝

single_turn

文本 prompt 行 → rollout_dataset.jsonl → verl_config.yaml，配合 compute_score reward 入口完成 GRPO + vLLM 训练；适合稳定可复现的指令跟随预测训练。

🛠️

time_series_agent

原始 chat 消息 + agent_name: time_series_forecast_agent + Cast-R1 风格 timestamp/value ground truth；导出 agent_loop_config.yaml 由 verl AgentLoop 驱动多轮工具调用。

🎯

统一奖励适配

两种 workflow 共享统一入口；single-turn 主打 format / accuracy / mse，agentic 进一步引入 length / normalized_mse / change_point / season_trend。

六类 Cast-R1 风格工具

extract_basic_statistics

计算单变量观测序列的 count / mean / std / min / max / median / first / last 等基础统计量。

extract_within_channel_dynamics

分析趋势斜率、平均变化、波动率、峰值数与方向反转次数，刻画局部动力学。

extract_forecast_residuals

基于简单局部基线汇总残差均值、std、绝对均值与最近残差，提示模型短期偏差。

extract_data_quality

识别非有限值、零差段、重复时间戳，给出 nonfinite_ratio、dropout_ratio 等数据质量指标。

extract_event_summary

基于差分阈值检测突变事件，返回事件段数量与代表性事件的时间戳/数值。

predict_time_series

调用配置好的 HTTP 模型服务（chronos2 / arima / patchtst / itransformer），失败时本地 ARIMA → last-value 兜底，输出与时间戳对齐的候选预测。

最小 Agentic Recipe 片段

rollout:
  workflow:
    name: time_series_agent
    max_steps: 3
    max_parallel_calls: 5
    tool_parser_format: hermes
    model_service_url: http://localhost:8994
    prediction_models: [chronos2, arima, patchtst, itransformer]
    local_fallback: arima_then_last_value

training:
  rewards:
    - name: format
    - name: length
    - name: normalized_mse
    - name: change_point
    - name: season_trend

Agentic workflow 通过 agent_loop_config.yaml 与 verl 原生 AgentLoop 对接， 模型最终以 <answer>YYYY-MM-DD HH:MM:SS value</answer> 行式输出预测， 由 timestamp/value parser 与对应奖励函数闭环验证。

典型使用工作流

CastFactory 的 CLI 入口 python -m castfactory.cli.run 接收一份 YAML recipe 并按 experiment.stage 自动调度 CPT / SFT / RLVR；CPT 与 SFT 走本地 Transformers backend，RLVR 走 verl 0.7.1（CastFactory 准备好 dataset、verl_config、reward 入口与启动命令）。 所有命令都支持 dotlist 覆盖，便于扫超参：

1. 安装

python -m pip install -e .                # 核心
python -m pip install -e ".[hf]"          # 可选：HuggingFace 模型加载
python -m pip install -e ".[train]"       # 可选：本地 CPT/SFT 训练栈
python -m pip install -e ".[agentic]"     # 可选：time-series agentic RL 工具

2. 运行 CPT

python -m castfactory.cli.run examples/cpt/etth1_qwen_cpt.yaml --mode fit
# 输出：./checkpoints/etth1_ot_qwen_cpt

3. 运行 SFT（接 CPT checkpoint）

python -m castfactory.cli.run examples/sft/etth1_qwen_sft.yaml --mode fit
# 自动读取 init_checkpoint: ./checkpoints/etth1_ot_qwen_cpt
# 输出：./checkpoints/etth1_ot_qwen_sft

4. 运行 RLVR（导出 verl 0.7.1 训练产物）

python -m castfactory.cli.run examples/rlvr/etth1_qwen_grpo.yaml --mode fit
# 在 ./checkpoints/etth1_ot_qwen_grpo/rlvr/ 下生成：
#   rollout_dataset.jsonl
#   verl_config.yaml
#   agent_loop_config.yaml   (仅 time_series_agent workflow)
#   launch_command.txt
# Reward 入口：castfactory.training.verl_reward_adapter:compute_score

5. dotlist 覆盖（按需扫超参）

python -m castfactory.cli.run examples/sft/etth1_qwen_sft.yaml --mode fit \
  model.backbone.model_name=Qwen/Qwen2.5-1.5B \
  training.args.num_train_epochs=1 \
  training.args.learning_rate=2.0e-4

6. 评测

python -m castfactory.cli.run examples/sft/etth1_qwen_sft.yaml --mode evaluate
# 写出 trace artifacts：
#   recipe.json · metrics.json · predictions.* · leaderboard · report.md

Recipe 示例（SFT）

每次实验对应一份完整 YAML recipe：

experiment:
  name: etth1_ot_qwen_sft
  stage: sft
  seed: 42

data:
  reader:
    name: csv
    path: ./castfactory/dataset/ETTh1/ETTh1.csv
    timestamp_col: date
    target_channels: [OT]
  split: { type: ratio, ratios: [0.7, 0.1, 0.2] }
  window: { context_length: 96, prediction_length: 96, stride: 96 }

representation:
  name: markdown_table
  significant_digits: 2

model:
  backbone:
    name: hf_causal_lm
    model_name: Qwen/Qwen2.5-1.5B
    trust_remote_code: true
  peft:
    method: lora
    r: 16
    lora_alpha: 32
    target_modules: [q_proj, v_proj]

training:
  init_checkpoint: ./checkpoints/etth1_ot_qwen_cpt
  checkpoint_dir: ./checkpoints/etth1_ot_qwen_sft
  backend:
    name: transformers
    max_prompt_length: 2048
    max_response_length: 1024
  args:
    num_train_epochs: 1
    learning_rate: 2.0e-4

阶段性建设路线

CastFactory 采用渐进式发布策略，已经按版本完成核心 CPT-SFT-RLVR 链路与 Agentic RLVR 集成， 后续将持续扩展骨架、表示与基准设施：

  • v0.1
    SFT + Evaluation MVP（已发布） recipe-centric 实验骨架、CSV reader、滑窗与 leakage 检查、SFT 数据集与 Transformers backend、JSON forecast parser，以及 standard / rolling / zero_shot 三种评测协议。
  • v0.2
    CPT 支持（已发布） CPTDataset + TransformersCPTBackend 完成 next-token 训练管线，FSDP 多卡，与 SFT 共享同一份 recipe schema。
  • v0.3
    RLVR 支持（已发布） Time-R1 风格 single-turn rollout、verl 0.7.1 GRPO + vLLM 配置、format / accuracy / mse / calibration / reasoning 五类可验证奖励、统一 reward 入口 compute_score。
  • v0.4
    Agentic RLVR（已发布） Cast-R1 风格 time_series_agent workflow、6 类时序分析与预测工具、verl 原生 AgentLoop 配置导出，以及 length / normalized_mse / change_point / season_trend 四种 agentic 奖励。
  • v0.5+
    Benchmark & Model Zoo（建设中） 沉淀更多数据集、horizon、表示与 base model 组合的 recipe，构建标准 benchmark 配置与跨实验 leaderboard 比较基础设施，逐步扩展为 CastFactory Model Zoo。

研究定位与贡献

四个层面的研究贡献

  • 1
    提出 CPT-SFT-RLVR 三阶段 LLM-based TSF 训练范式 明确区分时序适应、指令预测和可验证奖励优化三个阶段，建立 LLM-based TSF 的系统性训练方法论，并在一份 YAML recipe 内统一表达。
  • 2
    提供统一的时间序列到 LLM 训练样本的构建框架 统一 CSV reader、timestamp/ratio split、rolling windows、leakage 检查与 7 种内置时序表示，消除数据构建碎片化；外部 instruction 模板与丰富占位符让 prompt 工程可版本化。
  • 3
    构建面向 TSF 的可验证 RL 与 Agentic 训练接口 同时支持 single-turn GRPO 与 verl 原生 AgentLoop 两种 rollout workflow；将格式有效性、点预测误差、季节性/趋势/突变点结构转化为奖励，使 LLM 微调目标贴近真实预测指标，超越 token-level 交叉熵。
  • 4
    形成可复现的 LLM-based TSF 实验基础设施 通过 recipe 快照、metrics、predictions、leaderboard、report 与 stage metadata 等 trace artifacts 自动归档，支撑 Time-R1 与 Cast-R1 两条研究路线在同一框架内迭代。

现有 LLM-based 时间序列预测研究通常依赖临时性的数据序列化方式、指令构造方式和微调方案。 CastFactory（铸星坊）以 recipe 为唯一契约，提供一个统一的 CPT-SFT-RLVR 三阶段微调训练框架， 系统化标准时间序列数据构建、LLM 时序适应、指令跟随预测、可验证奖励优化、Agentic 工具调用与预测评测流程。

Existing LLM-based time series forecasting studies often rely on ad-hoc data serialization, instruction construction, and fine-tuning recipes. CastFactory provides a recipe-centric, unified CPT-SFT-RLVR fine-tuning framework that standardizes temporal data construction, LLM adaptation, instruction-following forecasting, verifiable reward optimization, agentic tool use, and forecasting evaluation.

项目边界声明： CastFactory（铸星坊）专注于 LLM-based TSF 微调训练框架本身，不重复造 verl 运行时—— RLVR 阶段以"准备 dataset、verl_config、reward 入口与启动命令"的方式与外部 verl 0.7.1 环境对接。 它首先是一个面向 LLM-based TSF 的模型工厂， 帮助研究者用统一流程回答核心问题： 给定一个基础 LLM 和一个时间序列预测任务，如何通过 CPT、SFT 和 RLVR 三阶段训练， 稳定、可复现地构建一个 LLM-based TSF Model？
