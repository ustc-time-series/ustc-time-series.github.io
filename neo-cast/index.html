NeoCast
时间序列预测研究操作系统

NeoCast 面向时间序列预测小模型研发，把“研究假设、模型配方、受控执行、实验评测、结果归因、研究记忆”串成可复现的自主研究闭环。它不是聊天助手，也不是无约束代码代理，而是让 Agent 在明确协议和搜索空间内持续产生、验证并沉淀可用时序模型的研究系统。

Positioning

把时序小模型研发变成可执行、可审计、可积累的研究流程

时间序列预测模型生态高度碎片化，从 Autoformer、PatchTST 到 iTransformer，每个新模型都伴随大量结构试错、训练策略调节和重复实验。NeoCast 的目标是把这些高频研究动作收束为受控闭环，让系统持续寻找“什么样的数据任务上，什么样的结构、目标和训练配方更有效”。

核心主张

NeoCast 以 TaskSpec 固定任务协议，以 CandidateRecipe 约束模型搜索空间，以 Runner 和 Evaluator 管理实验执行与晋级，再由 Memory Service 保存可复现的研究增量。系统优先服务小模型研发，优化目标同时包括精度、参数量、延迟、稳定性和复现性。

系统边界

  • 不是开放式文献综述 Agent，第一阶段聚焦 forecasting。
  • 不是自由无限制代码生成，优先使用 registry / DSL 表达候选。
  • 不是单纯 AutoML，实验结果会进入谱系、归因和跨任务记忆。
  • 不以自动写论文为主目标，优先产出可复现模型、代码、报告和经验库。
Research Loop

从假设到 Champion 的闭环研究流程

每一轮研究都围绕“可验证增量”展开：先提出小而明确的结构或训练假设，再生成候选配方、执行受控实验、比较 champion，并把有效和无效模式都写入研究记忆。

诊断任务

读取 TaskSpec，绑定数据版本，提取趋势、季节性、变量相关、shift、缺失率和 horizon/context 比等任务指纹。

建立基线

运行统计、NLinear、MLP、PatchTST 或 iTransformer 等 baseline，形成初始 champion 和误差结构。

生成配方

基于 TaskFingerprint 和历史记忆，提出若干 CandidateRecipe，并解释每个 action 的预期收益。

受控执行

通过 Static Validator、smoke run、低保真实验和完整验证筛掉非法、崩溃或明显无效候选。

晋级判断

按 val_avg_mse、参数量、延迟、稳定性、代码复杂度和失败率综合比较候选与 champion。

归因分析

分析提升来自 trend 拟合、长 horizon、变量依赖、正则化还是训练预算，并标记失败原因。

写入记忆

保存 recipe hash、patch diff、指标、资源消耗、verdict 和 analysis_summary，维护 lineage tree。

继续迭代

新一轮候选从 champion、Pareto front 和相似任务经验出发，避免重复试错并保留有效增量。

Architecture

七层架构：先固定协议，再释放智能

NeoCast 的关键不是让 Agent 更自由，而是在清晰边界内让 Agent 更可靠。系统把人类控制、Agent 编排、搜索空间、研究协议、执行环境、评测逻辑和研究记忆分层，以便每次实验都能被审计和复现。

Human Control Layer定义任务、批准搜索空间、冻结测试集、审核 champion、触发 blind test 与回滚。
Agent Orchestration LayerResearch Manager 协调 Diagnostic、Design、Coder、Runner、Analyst、Librarian 等角色。
Search Space LayerCandidateRecipe DSL、action registry、loss DSL 与 trainer DSL 决定 Agent 能探索什么。

Evaluation Layer多阶段筛选、指标计算、误差诊断、Promotion Gate 和 Metrics DB 支撑晋级判断。
Memory Layer保存实验谱系、成功/失败模式、任务指纹、patch diff、champion 和跨任务 meta memory。

Evaluator计算统一指标、误差分解、晋级判定和报告摘要。
Memory Service保存谱系、相似任务召回和跨任务经验迁移。
Governance Service保护 blind test、审批流、权限隔离与审计日志。
CandidateRecipe

用模块级配方描述候选模型，而不是直接让 Agent 任意改仓库

NeoCast 将候选空间从“选择某个论文模型族”升级为 TimeRecipe 风格的模块组合：Agent 按固定 pipeline 选择可注册 action，runtime 再把 CandidateRecipe 编译成可运行模型。这样既保留创新空间，也能审计每一次结构变化。

Input→ Preprocessing→ Embedding→ Backbone→ Fusion→ Head→ Training Strategy→ Forecast
Data Bias

TaskFingerprint 驱动先验

从 trend、seasonality、cross-variable correlation、distribution shift、missing ratio 与 horizon/context 比中提取任务偏置，用于候选排序和解释。

Structure

Action Registry 控制搜索边界

预处理覆盖 instance norm、series decomposition、differencing；embedding 覆盖 none、token、patch、invert、frequency；backbone 覆盖 linear、MLP、TCN、Transformer、SSM、Mixer 等。

Training

训练策略进入配方

Loss、optimizer、scheduler、regularization、early stopping、fidelity budget 和 distillation 都作为可审计字段，而不是散落在自由代码中。

Agent Orchestration

一个主控 Agent 加多个工具型角色，服务同一条研究链路

MVP 阶段可以先用单 Agent 和固定工具调用验证闭环；完整系统中，Research Manager 维护全局状态，其他 Agent 分别承担诊断、设计、编码、运行、分析和记忆管理。

Main Controller

Research Manager

读取任务规格、维护当前 champion 与预算状态，决定每轮研究目标，调度子角色，并根据实验结果更新下一轮搜索方向。

Diagnostic

任务画像

提取序列数量、缺失率、趋势、季节性、突变、变量相关、上下文与预测窗口比例等任务指纹。

Design

候选设计

基于任务指纹和记忆，选择 preprocessing、embedding、backbone、fusion、head 与 training action。

Coder

受限实现

把 CandidateRecipe 转成配置或模板化 patch，并执行静态检查、接口测试和预算检查。

Runner

实验执行

提交训练、跟踪状态、捕获异常、写入实验记录，并隔离 reference test 与 privileged log。

Analyst

结果归因

判断候选是真提升还是偶然波动，输出 horizon、序列 bucket、峰值段和训练稳定性的误差诊断。

Librarian

研究记忆

维护谱系树、相似任务召回、失败模式去重，把实验经验沉淀为跨任务可迁移规则。

Evaluation & Governance

避免验证集过拟合，保护测试集和研究可信度

NeoCast 的评测协议默认把日常决策信号、参考测试和最终盲测分离。Agent 高频看到的是验证集指标，reference test 由 Runner 或 Human Gate 在晋级节点查询，blind test 则作为最终 champion 的受限确认。

三层评测协议

  • L1 Dev Validation 用于每轮 keep / discard 决策。
  • L2 Reference Test 或 Shadow Holdout 只在晋级节点查询。
  • L3 Blind Test 用于最终 champion 确认和科研级报告。

时序安全原则

  • 归一化、填补和统计特征拟合只允许使用 train fold。
  • 未来不可见特征不能泄漏到训练样本。
  • Agent workspace 不挂载 reference test 标签或 privileged promotion log。

晋级与回滚

  • 候选必须满足精度、参数量、延迟、稳定性和复现约束。
  • 维护 Pareto front，分别保留精度、延迟、参数量和综合最优方案。
  • 每个 champion 都支持回放实验、导出 patch diff 与回滚父版本。
Roadmap

从 Phase 0a 闭环验证，到跨任务元记忆和盲测治理

NeoCast 的工程路线强调逐步收紧协议：先让系统跑通 keep / discard，再冻结 CandidateRecipe v1，之后扩展 registry、低保真筛选、多 seed、Champion / Challenger、跨任务记忆和 blind test 服务。

Milestone 0

基础底座与最小闭环

实现 prepare_tsf.py、train.py baseline、program.md、TaskSpec / EvalSpec / BudgetSpec，并完成第一次自主实验闭环。

Milestone 1

Recipe 化受限研究闭环

引入 action registry、build_model(candidate_recipe)、Static Validator、Dataset Profiler、低保真与完整训练流水线。

Milestone 2

TimeRecipe 风格模块组合搜索

扩展 preprocessing、embedding、backbone、fusion、prediction head 和 Loss grammar，形成可组合的小模型结构创新空间。

Milestone 3

多 seed、Champion / Challenger 与研究记忆

实现多 seed、Pareto front、Utility 评分、Shadow Holdout 查询和 Memory Service v1。

Milestone 4-6

跨任务元记忆、治理与 P2 扩展

构建相似任务检索和 recipe rank predictor，引入 blind test 服务、审计审批、概率预测任务和完整蒸馏模式。

研究效果相对 baseline 的 val_avg_mse 提升、Pareto 改进数量、top-k recipe 命中率。

知识积累任务级记忆、跨任务 meta memory 和可迁移 recipe prior 的质量。
