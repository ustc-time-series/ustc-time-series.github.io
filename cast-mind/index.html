一句话定位： CastMind（星思）是一个时间序列认知大模型。 它不仅学习时间序列中的数值模式，还进一步理解序列所处的情境上下文， 分析趋势变化背后的原因，并通过慢思考推理与记忆交互不断获取证据、修正预测判断， 从而实现更可靠、更可解释和更具适应性的时序预测。 模型沿着 CastMind-Base → CastMind-SFT → CastMind-CoT → CastMind-Agentic Reasoning 的四阶段路线，逐步建立 "感知与理解—情境推理—慢思考推理—记忆交互—持续进化"五项核心能力。

项目简介

CastMind（星思）是一类面向复杂时序场景的新型基础模型。 传统时序模型局限于从历史数值中提取模式，而 CastMind（星思）的目标更进一步： 让模型能够理解序列所处的情境上下文——包括 Meta Data、外生变量、 协变量、领域描述与事件信息——并在此基础上分析趋势变化背后的原因， 理解天气变化如何影响电力负荷，手术中的生理指标如何预示术后风险， 节假日与政策事件如何驱动交通和消费行为。

CastMind（星思）的关键范式是将 Search-o1 式的"边思考边检索" 扩展到时间序列预测：模型不是一次性输出预测，而是在慢思考过程中 围绕趋势变化、异常原因、未来演化路径与不确定性展开分步分析， 动态发现证据缺口，主动与 Memory 环境交互—— 检索历史相似模式、领域规律、失败经验与过往数据分析结论—— 并据此持续补充证据、修正预测判断， 实现从数值拟合到情境认知推理的范式跨越。

五大核心能力

时间序列认知大模型围绕复杂真实时序场景，构建从感知、推理到自主进化的五大核心能力：

👁️

感知与理解

Perception & Understanding

模型能够理解原始时间序列的趋势、周期、波动、异常、突变等结构特征， 同时理解 Meta Data、外生变量、协变量、领域描述与事件信息等多源情境上下文， 形成对序列本身与其所处环境的整体认知。

🔗

情境推理

Contextual Reasoning

模型能够分析情境因素如何影响未来趋势：天气变化如何驱动电力负荷， 手术中的生理指标变化如何预示术后风险，节假日与政策事件如何塑造 交通流量和消费行为——从情境原因出发，推断趋势走向。

🧠

慢思考推理

Slow Thinking Reasoning

模型不是一次性输出预测，而是围绕趋势变化、异常原因、未来演化路径与 不确定性展开分步分析，在长思维链中逐步构建预测判断， 类比 Search-o1 的"边思考边推理"范式。

🗄️

记忆交互

Memory Interaction

模型可以与 Memory 环境交互，在慢思考过程中主动检索历史相似模式、 领域规律、失败经验与过往数据分析结论，将检索证据融入推理上下文， 并据此持续补充证据、修正预测判断。

🚀

持续进化

Continuous Evolution

模型通过 CPT、SFT、CoT、Agentic Reasoning 等阶段逐步提升时序理解、 预测推理和自主交互能力——从基础接入到任务适配，从慢思考推理到 Memory 驱动的自主修正，形成可持续演进的认知能力路线。

核心技术体系

CastMind（星思）的技术体系从模型、数据、算法三个层面构建， 三者相互支撑，共同实现慢思考时间序列推理：

Model Layer

Feature Encoder + LLM Backbone 架构

原始时间序列首先经过独立的 Time Series Feature Encoder 编码为 Temporal Embedding，再送入预训练语言模型 Backbone 进行推理。 Feature Encoder 包含多尺度 patch 投影（soft temporal tokens）与 量化语义通路（趋势 / 周期 / 异常 temporal tokens）两条支路， 将连续数值序列转化为 LLM 可直接理解的结构化时序表示。 输出端结合 forecasting head 输出点预测、区间预测或概率分布。

Feature Encoder · Temporal Embedding · LLM Backbone · Forecasting Head
Data Layer

情境感知时间序列训练 Instance

训练数据不是普通时间序列窗口，而是情境感知的预测 instance， 包含历史序列、多模态情境、领域 metadata、预测指令、未来目标、 推理轨迹、Memory 交互轨迹和奖励信号。 训练语料来自能源、交通、气象、医疗、金融、工业等多个领域， 通过多领域、多频率、多变量混合训练获得跨域泛化能力。 FutureCast（天星台） 作为情境感知训练语料与 out-of-domain 评测底座。

多领域混合 · FutureCast（天星台） 底座 · Reasoning Trace · Memory Trace
Algorithm Layer

CastMind-Base · CastMind-SFT · CastMind-CoT · CastMind-Agentic Reasoning

CastMind（星思）不被简单拆成 Base / SFT / RL，而是按能力递进组织为四个阶段： Base 解决"如何接入时间序列"，SFT 解决"如何完成时序任务"， CoT 解决"如何进行慢思考预测"，Agentic Reasoning 解决 "如何在推理过程中自主获取证据并修正判断"。 最后阶段强调 Memory-interactive reasoning，而不是简单工具调用。

CastMind-Base → CastMind-SFT → CastMind-CoT → CastMind-Agentic Reasoning

训练 Instance 结构

一个典型的 CastMind（星思）训练样本包含以下八类信息：

Instance = {
  historical_time_series ← 历史数值模式
  multimodal_context ← 外部情境证据（天气、事件、日历、空间…）
  domain_metadata ← 领域、频率、变量、单位信息
  forecasting_instruction ← 预测任务定义
  future_target ← 未来真实值（监督信号）
  reasoning_trace ← 慢思考推理过程（SFT / CoT 阶段学习）
  memory_interaction_trace← Memory 检索、证据利用与预测修正轨迹
  reward_signals ← 结果奖励 + 过程奖励 + Memory 交互反馈
}

CastMind-Base → CastMind-SFT → CastMind-CoT → CastMind-Agentic Reasoning

CastMind（星思）采用四个能力递进阶段： CastMind-Base → CastMind-SFT → CastMind-CoT → CastMind-Agentic Reasoning。 Base 阶段解决"如何接入时间序列"；SFT 阶段解决"如何完成时序任务"； CoT 阶段解决"如何进行慢思考预测"；Agentic Reasoning 阶段解决 "如何在推理过程中自主获取证据并修正判断"。

Stage 1 · CastMind-Base

基础模型阶段
接入时间序列信号

输入：原始时间序列 + 多模态情境 + 跨域语料

核心目标是建立"时间序列特征编码 + LLM Backbone"的模型底座。 原始时间序列首先经过 Time Series Feature Encoder 转换为 Embedding， 再输入语言模型，使模型具备跨域时间序列表征、基础预测、上下文融合和 多模态情境对齐能力。这个阶段重点不是复杂推理，而是让语言模型真正接入时间序列信号。

输出：CastMind-Base
Stage 2 · CastMind-SFT

任务适配阶段
按任务分析和预测

输入：监督微调任务样本 + 预测与解释示范

核心目标是让模型从"能表示时间序列"进一步变成 "能按照任务要求分析和预测时间序列"。 通过监督微调，模型学习趋势分析、预测生成、异常解释、外生变量理解、 Meta Data 融合、领域描述理解等能力，解决指令跟随、任务格式、预测输出和基础解释问题。

输出：CastMind-SFT
Stage 3 · CastMind-CoT

慢思考推理阶段
推理式预测

输入：长思维链时序推理样本 + 推理质量反馈

核心目标是强化模型的长思维链时序推理能力。 模型不只是直接输出预测结果，而是能够围绕趋势变化、周期波动、异常扰动、 事件影响、情境变化和未来演化路径展开分步分析。 这个阶段把时间序列预测从"模式外推"推进到"推理式预测"。

输出：CastMind-CoT
Stage 4 · CastMind-Agentic Reasoning

自主交互推理阶段
Memory-interactive reasoning

输入：Memory 环境交互 + 证据检索与修正轨迹

核心目标是让 CastMind（星思）具备类似 Search-o1 的边思考边检索能力。 模型在慢思考过程中可以主动发现证据缺口，与 Memory 环境交互， 检索历史相似模式、领域知识、失败经验、数据分析结论和过往预测轨迹， 并基于新证据持续修正中间判断和最终预测。这个阶段强调 Memory-interactive reasoning， 不是简单工具调用。

输出：CastMind-Agentic Reasoning

CastMind-Agentic Reasoning 的三类策略优化

  • 1
    Memory Seeking Policy · 记忆检索策略 模型学习何时主动检索 Memory，以及应该检索哪类 Memory：相似历史片段、历史预测经验、领域规律、异常模式、失败案例、数据分析报告等。关键不是"有没有 Memory"，而是模型能否在推理中动态识别证据缺口。
  • 2
    Evidence Grounding Policy · 证据利用策略 模型检索到 Memory 后，需要判断哪些证据有用、哪些证据会误导预测，并将证据合理融入慢思考推理链，而不是简单拼接到上下文中。这能减少 Memory-RAG 式幻觉，也让推理过程更可解释。
  • 3
    Forecast Revision Policy · 预测修正策略 模型在获得新证据后学习更新中间判断和最终预测。例如初始判断未来负荷会上升，但 Memory 显示相似天气条件下历史负荷反而下降时，模型需要重新评估原始假设并形成新的预测路径。

四阶段能力路线

CastMind（星思）的四个阶段不是简单的 Base / SFT / RL 分段， 而是明确对应四类能力跃迁：

CastMind-Base

接入时间序列

基础模型阶段

建立 Time Series Feature Encoder + LLM Backbone 的模型底座。 原始时间序列被转换为 Embedding 后输入语言模型， 让模型具备跨域时序表征、基础预测、上下文融合和多模态情境对齐能力。

CastMind-SFT

完成时序任务

任务适配阶段

通过监督微调学习趋势分析、预测生成、异常解释、外生变量理解、 Meta Data 融合和领域描述理解，解决指令跟随、任务格式、预测输出和基础解释能力。

CastMind-CoT

慢思考预测

慢思考推理阶段

强化长思维链时序推理能力，围绕趋势变化、周期波动、异常扰动、 事件影响、情境变化和未来演化路径展开分步分析， 将预测从模式外推推进到推理式预测。

CastMind-Agentic Reasoning

自主获取证据并修正判断

自主交互推理阶段

在慢思考过程中主动发现证据缺口，与 Memory 环境交互， 检索历史相似模式、领域知识、失败经验、数据分析结论和过往预测轨迹， 并基于新证据持续修正中间判断和最终预测。

这一路线可以概括为： Base 阶段解决"如何接入时间序列"；SFT 阶段解决"如何完成时序任务"； CoT 阶段解决"如何进行慢思考预测"；Agentic Reasoning 阶段解决 "如何在推理过程中自主获取证据并修正判断"。

评估体系

CastMind（星思）的评估同时覆盖 in-domain 能力和 out-of-domain 泛化能力， 全面验证慢思考时序推理模型的综合性能：

In-domain 评估

已见领域基础预测能力

使用现有数据集测试集，检验 CastMind（星思）在已见领域和数据分布下的性能。

  • 点预测指标：MAE · MSE · RMSE · sMAPE · MASE
  • 概率预测指标：CRPS · Pinball Loss · Prediction Interval Coverage
  • 推理质量指标：趋势解释合理性 · 情境归因一致性 · 预测—推理一致性

在四个阶段（Base / SFT / CoT / Agentic Reasoning）间进行对比，展示各阶段对预测、推理与记忆交互能力的增益。

Out-of-domain 评估

跨领域情境感知泛化能力

以 FutureCast（天星台） 为核心评测集，构造未见领域、未见上下文类型和复杂情境变化下的预测任务。

  • 能否适应新的领域和采样频率
  • 能否理解未见的上下文情境类型
  • 能否在外部事件、分布漂移、非平稳变化和信息不完整下给出稳健预测
  • 能否通过慢思考推理解释未来趋势变化
  • 能否输出合理的不确定性估计
