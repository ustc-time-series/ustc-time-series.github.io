项目简介

CastMind（星思）是一类面向复杂时序场景的新型基础模型。 它不把数值序列简单文本化，而是以 Feature Encoder 原生建模动态数值序列， 将连续变化、局部波动、周期结构、突变和异常转化为可被 LLM 使用的时序表示。 在此基础上，CastMind（星思）进一步融入 Context-aware Feature——包括 Meta Data、外生变量、协变量、领域描述、事件信息和环境状态——让预测建立在序列本身 与其所处情境的联合理解上。

CastMind（星思）的关键范式是 Agentic Reasoning： 模型不是一次性输出预测，而是在推理过程中动态识别证据缺口， 与 Memory、检索器、外部变量服务和任务环境进行自主多轮交互。 它可以检索历史相似模式、领域规律、失败经验和过往数据分析结论， 再把新证据纳入上下文推理，持续修正中间判断和最终预测， 实现从数值拟合到情境认知与自主决策的范式升级。

三大核心亮点

CastMind（星思）的页面表达聚焦三项差异化能力：原生时序特征编码、情境感知特征融合和可与环境交互的 Agentic Reasoning。

TS

Feature Encoder 原生建模动态数值序列

Feature-native Temporal Modeling

动态数值序列不再被直接压平成文本 token。CastMind（星思）通过 Feature Encoder 原生建模连续值、局部波动、周期结构、突变点和异常模式，形成 dynamic_numeric_sequence 表示，再交给 LLM Backbone 进行预测和推理。

CTX

情境融入：Context-aware Feature

Contextual Integration

每个预测 instance 同时包含可见的上下文特征：metadata、外生变量、协变量、事件、日历、 领域描述和环境状态。模型学习如何把 context_aware_features 与时序表示对齐，从情境原因解释趋势变化。

AGT

Agentic Reasoning：自主多轮交互与上下文推理

Agentic Reasoning

模型可以在预测过程中主动判断缺失证据，调用 Memory、检索器、上下文工具或外部变量服务， 形成 agentic_interaction_trace。重点不是一次工具调用，而是可评估的自主多轮交互、 上下文推理和预测修正能力。

核心技术体系

CastMind（星思）的技术体系从模型、数据、算法三个层面构建， 分别对应 Feature Encoder、Context-aware Feature 与 Agentic Reasoning 三项亮点：

Model Layer

Feature Encoder 原生动态序列建模

原始时间序列首先经过独立的 Feature Encoder， 在数值空间内建模连续变化、局部波动、周期结构、突变点和异常。 Encoder 输出 soft temporal tokens 与结构化 temporal embeddings， 再送入预训练语言模型 Backbone。这样模型接收的是原生时序表示， 而不是把数值强行转写成自然语言片段。

Feature Encoder · dynamic_numeric_sequence · Temporal Embedding · Forecasting Head
Data Layer

Context-aware Feature 训练 Instance

训练数据不是普通时间序列窗口，而是情境感知预测 instance： 历史序列之外，还包含 metadata、协变量、外生变量、事件、领域描述、 可见性边界和环境状态。Context-aware Feature 明确回答 “当前预测发生在什么情境中、哪些上下文可以被模型合法使用”。 FutureCast（天星台） 可作为情境感知训练语料与 out-of-domain 评测底座。

context_aware_features · Metadata · Events · Exogenous Signals · Visibility
Algorithm Layer

Agentic Reasoning 与环境多轮交互

CastMind（星思）按能力递进组织为 Base、SFT、Structured Reasoning 与 Agentic Reasoning 四个阶段。最后阶段训练模型在预测过程中识别证据缺口、 与 Memory、检索器、上下文工具和外部变量服务进行自主多轮交互， 再基于环境反馈更新上下文推理和预测判断。

Agentic Reasoning · Environment State · Multi-turn Interaction · Forecast Revision

训练 Instance 结构

一个典型的 CastMind（星思）训练样本包含以下八类信息：

Instance = {
  dynamic_numeric_sequence ← Feature Encoder 原生建模的动态数值序列
  context_aware_features ← 外部情境证据（天气、事件、日历、空间、环境状态…）
  domain_metadata ← 领域、频率、变量、单位信息
  forecasting_instruction ← 预测任务定义
  future_target ← 未来真实值（监督信号）
  reasoning_trace ← 结构化上下文推理过程（SFT / Reasoning 阶段学习）
  agentic_interaction_trace← 环境交互、证据获取与预测修正轨迹
  reward_signals ← 结果奖励 + 推理过程奖励 + Agentic 交互反馈
}

四阶段训练范式

CastMind（星思）采用四个能力递进阶段： CastMind-Base → CastMind-SFT → CastMind-Structured Reasoning → CastMind-Agentic Reasoning。 Base 阶段解决"如何以 Feature Encoder 接入动态数值序列"；SFT 阶段解决"如何融合 Context-aware Feature 完成时序任务"； Structured Reasoning 阶段解决"如何进行上下文推理预测"；Agentic Reasoning 阶段解决 "如何与环境自主多轮交互、获取证据并修正判断"。

Stage 1 · CastMind-Base

基础模型阶段
接入时间序列信号

输入：原始时间序列 + 多模态情境 + 跨域语料

核心目标是建立"Feature Encoder + LLM Backbone"的模型底座。 原始时间序列首先经过 Feature Encoder 在数值空间中建模动态变化， 再输入语言模型，使模型具备跨域时间序列表征、基础预测、上下文融合和 多模态情境对齐能力。这个阶段重点不是复杂推理，而是让语言模型真正接入动态数值序列信号。

输出：CastMind-Base
Stage 2 · CastMind-SFT

任务适配阶段
按任务分析和预测

输入：监督微调任务样本 + 预测与解释示范

核心目标是让模型从"能表示时间序列"进一步变成 "能按照任务要求分析和预测时间序列"。 通过监督微调，模型学习趋势分析、预测生成、异常解释、外生变量理解、 Meta Data 融合、领域描述理解和 Context-aware Feature 对齐， 解决指令跟随、任务格式、预测输出和基础解释问题。

输出：CastMind-SFT
Stage 3 · CastMind-Structured Reasoning

上下文推理阶段
推理式预测

输入：结构化时序推理样本 + 推理质量反馈

核心目标是强化模型的结构化上下文推理能力。 模型不只是直接输出预测结果，而是能够围绕趋势变化、周期波动、异常扰动、 事件影响、情境变化和未来演化路径展开分步分析。 这个阶段把时间序列预测从"模式外推"推进到"推理式预测"。

输出：CastMind-Structured Reasoning
Stage 4 · CastMind-Agentic Reasoning

Agentic Reasoning 阶段
环境自主多轮交互

输入：环境状态 + Memory/工具交互 + 证据检索与修正轨迹

核心目标是让 CastMind（星思）具备可评估的 Agentic Reasoning 能力。 模型在预测过程中可以主动发现证据缺口，与 Memory、检索器、外部变量服务和任务环境 自主多轮交互，检索历史相似模式、领域知识、失败经验、数据分析结论和过往预测轨迹， 并基于新证据持续修正上下文判断和最终预测。这个阶段强调环境交互闭环， 不是一次性的工具调用展示。

输出：CastMind-Agentic Reasoning

CastMind-Agentic Reasoning 的三类策略优化

  • 1
    Interaction Seeking Policy · 交互触发策略 模型学习何时需要与环境交互，以及应该访问哪类环境：Memory、检索器、上下文工具、外部变量服务或任务状态。关键不是"有没有工具"，而是模型能否在推理中动态识别证据缺口。
  • 2
    Context Grounding Policy · 上下文证据利用策略 模型获得环境反馈后，需要判断哪些证据有用、哪些证据会误导预测，并将证据合理融入上下文推理，而不是简单拼接到输入中。这能减少检索噪声，也让预测依据更可解释。
  • 3
    Forecast Revision Policy · 预测修正策略 模型在获得新证据后学习更新中间判断和最终预测。例如初始判断未来负荷会上升，但环境反馈显示相似天气条件下历史负荷反而下降时，模型需要重新评估原始假设并形成新的预测路径。

四阶段能力路线

CastMind（星思）的四个阶段不是简单的 Base / SFT / RL 分段， 而是明确对应四类能力跃迁：

CastMind-Base

接入时间序列

基础模型阶段

建立 Feature Encoder + LLM Backbone 的模型底座。 原始动态数值序列被编码为 temporal embeddings 后输入语言模型， 让模型具备跨域时序表征、基础预测、上下文融合和多模态情境对齐能力。

CastMind-SFT

完成时序任务

任务适配阶段

通过监督微调学习趋势分析、预测生成、异常解释、外生变量理解、 Meta Data 融合、领域描述理解和 Context-aware Feature 对齐， 解决指令跟随、任务格式、预测输出和基础解释能力。

CastMind-Structured Reasoning

上下文推理预测

结构化推理阶段

强化结构化上下文推理能力，围绕趋势变化、周期波动、异常扰动、 事件影响、情境变化和未来演化路径展开分步分析， 将预测从模式外推推进到推理式预测。

CastMind-Agentic Reasoning

自主获取证据并修正判断

自主交互推理阶段

在预测过程中主动发现证据缺口，与 Memory、检索器、上下文工具、 外部变量服务和任务环境自主多轮交互，检索历史相似模式、领域知识、 失败经验、数据分析结论和过往预测轨迹，并基于新证据持续修正中间判断和最终预测。

这一路线可以概括为： Base 阶段解决"如何以 Feature Encoder 接入动态数值序列"；SFT 阶段解决"如何融合 Context-aware Feature 完成时序任务"； Structured Reasoning 阶段解决"如何进行上下文推理预测"；Agentic Reasoning 阶段解决 "如何与环境自主多轮交互、获取证据并修正判断"。

评估体系

CastMind（星思）的评估同时覆盖 in-domain 能力和 out-of-domain 泛化能力， 全面验证 Feature Encoder、Context-aware Feature 与 Agentic Reasoning 的综合增益：

In-domain 评估

已见领域基础预测能力

使用现有数据集测试集，检验 CastMind（星思）在已见领域和数据分布下的性能。

  • 点预测指标：MAE · MSE · RMSE · sMAPE · MASE
  • 概率预测指标：CRPS · Pinball Loss · Prediction Interval Coverage
  • Encoder 指标：动态数值序列表示质量 · 关键变化点捕获 · 趋势方向一致性
  • Context 指标：Context Attribution · 情境归因一致性 · 可见上下文利用率
  • Agentic 指标：交互触发准确性 · 环境证据利用率 · 预测修正有效性

在四个阶段（Base / SFT / Structured Reasoning / Agentic Reasoning）间进行对比，展示各阶段对预测、上下文推理与环境交互能力的增益。

Out-of-domain 评估

跨领域情境感知泛化能力

以 FutureCast（天星台） 为核心评测集，构造未见领域、未见上下文类型和复杂情境变化下的预测任务。

  • 能否适应新的领域和采样频率
  • 能否理解未见的 Context-aware Feature 类型
  • 能否在外部事件、分布漂移、非平稳变化和信息不完整下给出稳健预测
  • 能否通过上下文推理解释未来趋势变化
  • 能否在信息不完整时与环境自主多轮交互并修正预测
  • 能否输出合理的不确定性估计
