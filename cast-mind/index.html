CastMind 面向情境感知时间序列预测的慢思考基础模型

以预训练语言模型为基座，将连续时间序列、多模态情境上下文与推理任务指令统一建模—— 让模型不仅能预测未来数值，更能理解趋势变化的情境原因，并进行可验证的慢思考推理。

一句话定位： CastMind 是一个面向情境感知时间序列预测的 多模态慢思考基础模型—— 以预训练语言模型为基座，通过 CPT-SFT-RLVR 三阶段训练， 构建能够进行多模态情境感知、长思维链推理与可验证预测的时间序列推理模型。

项目简介

传统时间序列预测模型大多属于"快思考"范式，通过历史窗口直接拟合未来窗口， 本质上依赖模式识别和函数逼近。然而真实预测场景要求的不只是数值外推—— 模型还需要回答"当前序列处于什么状态""哪些情境因素正在影响未来趋势" "未来变化为什么会发生""预测结果是否可靠"。

CastMind 正是为回应这些更高层次的预测认知需求而构建。它连接 Time Series Foundation Model 与 LLM-based Reasoning Model， 将时间序列预测从传统的模式发现与数值拟合，推进到基于情境证据的 认知推理与慢思考预测。

三大核心特色

CastMind 从传统数值预测模型的能力边界出发，围绕真实预测场景中的三类核心需求展开：

🌐

多模态情境感知

Multimodal Context Awareness

现实时间序列并不孤立存在——能源负荷受天气与政策影响，交通流量受事件与天气影响， 医疗信号受干预与用药影响。CastMind 将预测 instance 定义为历史序列、 上下文情境、领域知识、任务目标与预测约束的整体， 让模型真正理解"情境如何影响趋势"，而不只是拼接更多协变量。

🧠

慢思考思维链推理

Slow-thinking Chain-of-Thought Reasoning

CastMind 引入类似大模型推理的 slow-thinking 能力，使模型围绕预测任务进行多步分析： 观察历史模式 → 分解趋势结构 → 识别关键情境 → 分析变化原因 → 推断未来方向 → 估计不确定性 → 自我验证。 对于非平稳序列、事件驱动型趋势变化和复杂预测任务，慢思考能力具有不可替代的意义。

🌍

跨领域通用泛化

Cross-domain Generalization

时间序列广泛存在于能源、交通、金融、气象、医疗、工业、零售和水文等多个领域。 CastMind 通过多领域混合训练，学习跨领域共享的时间模式、情境表征和预测推理能力， 目标是在未见的新领域、新数据集和新预测任务上展现强大的 zero-shot 或 few-shot 泛化能力。

核心技术体系

CastMind 的技术体系从模型、数据、算法三个层面构建， 三者相互支撑，共同实现慢思考时间序列推理：

Model Layer

以语言模型为核心的模型架构

以预训练语言模型为基础 Backbone，利用其指令理解、语义建模和长思维链推理能力。 通过 Temporal Token Bridge 将时间序列灌入 LLM： 连续表示通路将序列切分为多尺度 patch 并投影为 soft temporal tokens， 离散语义通路通过量化方法将片段转化为具有趋势、周期、异常语义含义的 temporal tokens。 输出端结合专门的 forecasting head 输出点预测、区间预测或概率分布。

Temporal Token Bridge · Forecasting Head · Verifier
Data Layer

情境感知时间序列训练 Instance

训练数据不是普通时间序列窗口，而是情境感知的预测 instance， 包含历史序列、多模态情境、领域 metadata、预测指令、未来目标、推理轨迹和验证信号。 训练语料来自能源、交通、气象、医疗、金融、工业等多个领域， 通过多领域、多频率、多变量混合训练获得跨域泛化能力。 FutureCast（天星台） 作为情境感知训练语料与 out-of-domain 评测底座。

多领域混合 · FutureCast（天星台） 底座 · Reasoning Trace
Algorithm Layer

CPT · SFT · RLVR 三阶段训练

三阶段训练使 CastMind 从语言模型出发， 依次获得时序感知能力、指令推理能力和可验证预测能力。 CPT 完成时序空间迁移，SFT 实现慢思考指令跟随， RLVR 通过预测误差、趋势方向、区间覆盖率等 verifier 构建奖励， 将慢思考推理与可验证预测结果真正连接。

CPT → SFT → RLVR · Verifiable Rewards

训练 Instance 结构

一个典型的 CastMind 训练样本包含以下七类信息：

Instance = {
  historical_time_series ← 历史数值模式
  multimodal_context ← 外部情境证据（天气、事件、日历、空间…）
  domain_metadata ← 领域、频率、变量、单位信息
  forecasting_instruction ← 预测任务定义
  future_target ← 未来真实值（监督信号）
  reasoning_trace ← 慢思考推理过程（SFT 学习）
  verification_signal ← 可验证奖励信号（RLVR 阶段）
}

CPT-SFT-RLVR 三阶段训练

CastMind 的三阶段训练依次赋予模型时序感知、指令推理与可验证优化三类能力， 每个阶段都有清晰的目标和产出：

Stage 1 · CPT

持续预训练
时序空间迁移

输入：大规模多领域时间序列语料

目标是让预训练语言模型获得时间序列感知能力。 训练目标包括未来 patch 预测、masked temporal modeling、 temporal token prediction 和 context-series alignment， 使模型完成从语言空间到时间序列空间的初步迁移， 学习趋势、周期、异常、波动、漂移和跨变量依赖。

输出：CastMind-Base
Stage 2 · SFT

监督微调
慢思考指令推理

输入：情境感知时间序列指令样本 + 推理轨迹

目标是让模型学会按指令完成时间序列预测、解释与推理。 SFT 数据涵盖趋势分析、周期判断、异常识别、情境归因、 不确定性分析、反事实预测和预测自验证等高质量指令样本。 CastMind 由此从时序感知模型转化为 能进行慢思考推理的时序指令模型。

输出：CastMind-Instruct
Stage 3 · RLVR

可验证奖励强化学习
预测质量优化

输入：预测结果 + 真实未来值 + 可验证奖励信号

时间序列预测天然具有可验证信号：预测误差、趋势方向正确性、 区间覆盖率、分布校准程度、情境一致性和鲁棒性。 RLVR 阶段基于这些 verifier 构建奖励函数， 是 CastMind 区别于普通 LLM-based TSF 的关键环节—— 将慢思考推理与可验证预测结果真正连接。

输出：CastMind-R1

模型版本体系

三个训练阶段对应三个递进版本，每个版本可独立评估，也可作为下一阶段的起点：

CastMind-Base

基础时序感知版本

CPT 阶段产出

完成时序空间迁移，具备对数值序列、趋势、周期和跨变量依赖的基础感知能力。 是 SFT 和 RLVR 的起点，也可用于评估持续预训练对时序建模的底层增益。

CastMind-Instruct

指令推理版本

CPT + SFT 阶段产出

能够按照指令完成情境感知预测、趋势推理、异常识别和不确定性分析， 具备慢思考推理能力和预测解释能力。 可直接用于复杂预测任务和人机协同推理场景。

CastMind-R1

可验证推理版本

CPT + SFT + RLVR 阶段产出

在指令推理能力基础上，通过可验证奖励进一步优化预测精度、推理一致性和鲁棒性。 预测结果更准确、更稳健、更可信，慢思考推理链与预测输出真正对齐。

三个版本可分别对应 in-domain 和 out-of-domain 评估， 系统展示 CPT、SFT、RLVR 各阶段对预测能力、推理能力和可验证性带来的逐步增益。

评估体系

CastMind 的评估同时覆盖 in-domain 能力和 out-of-domain 泛化能力， 全面验证慢思考时序推理模型的综合性能：

In-domain 评估

已见领域基础预测能力

使用现有数据集测试集，检验 CastMind 在已见领域和数据分布下的性能。

  • 点预测指标：MAE · MSE · RMSE · sMAPE · MASE
  • 概率预测指标：CRPS · Pinball Loss · Prediction Interval Coverage
  • 推理质量指标：趋势解释合理性 · 情境归因一致性 · 预测—推理一致性

在三个版本（Base / Instruct / R1）间进行对比，展示各训练阶段对预测与推理能力的增益。

Out-of-domain 评估

跨领域情境感知泛化能力

以 FutureCast（天星台） 为核心评测集，构造未见领域、未见上下文类型和复杂情境变化下的预测任务。

  • 能否适应新的领域和采样频率
  • 能否理解未见的上下文情境类型
  • 能否在外部事件、分布漂移、非平稳变化和信息不完整下给出稳健预测
  • 能否通过慢思考推理解释未来趋势变化
  • 能否输出合理的不确定性估计

项目定位

CastMind 在整个时间序列 AI 研究体系中扮演的角色：

  • 1
    连接基础模型与推理模型的新型模型形态 CastMind 既不是传统 TSF 模型（只做数值外推），也不是通用语言模型（不懂时序），而是两者的交汇——以 LLM 为基座，以时间序列预测为核心任务，以情境感知推理为核心能力。
  • 2
    CastFactory（铸星坊） 训练框架的旗舰模型 CastMind 通过 CastFactory（铸星坊） 的 CPT-SFT-RLVR 三阶段训练流程构建，是 CastFactory（铸星坊） 框架能力的具体体现和实验载体，两者在技术上形成完整闭环。
  • 3
    FutureCast（天星台） 基准的核心评测对象 FutureCast（天星台） 的情境感知评测任务专为测试 CastMind 这类模型而设计。CastMind 的 out-of-domain 能力通过 FutureCast（天星台） 进行系统验证，两者在数据与评测上形成配套关系。
  • 4
    推动时序预测从"快思考"到"慢思考"的范式探索 CastMind 的 slow-thinking reasoning 能力是对当前时间序列基础模型的重要扩展，旨在让预测模型真正具备情境理解、因果归因和不确定性感知能力，而不只是更大的数值外推器。

CastMind 是一个面向情境感知时间序列预测的多模态慢思考基础模型。 它以预训练语言模型为基座，将连续时间序列、多模态上下文、领域知识和预测任务指令统一建模， 旨在让模型不仅能够预测未来数值，还能够理解历史模式、识别关键情境、推理趋势变化、 估计预测不确定性，并对预测结果进行自我验证。

CastMind is a multimodal slow-thinking foundation model for context-aware time series forecasting, built upon pretrained language models and trained through CPT, SFT, and RLVR to reason over temporal patterns, contextual evidence, and future uncertainties across domains.
