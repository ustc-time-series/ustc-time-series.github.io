一句话定位： CastEmbedding 是一个面向情境感知时间序列分析的通用嵌入表示基础设施： 它不只学习历史曲线的 pattern similarity，而是学习给定情境、任务目标和未来演化语义约束下的 context-conditioned temporal semantics。

项目简介

时间序列嵌入表示是支撑时序检索、分类、聚类、预测迁移、近邻样本增强和智能体推理的重要基础能力。 传统时间序列表征学习通常以数值序列本身为核心，关注趋势、周期、波动、异常和局部形态， 主要回答“哪些时间序列在曲线形态上相似”。

但真实复杂系统中的时间序列并不是孤立存在的。相似的历史曲线可能因为天气、节假日、政策、 区域状态、临床阶段或设备工况等情境差异而走向不同未来；形态并不完全一致的序列， 也可能在相同情境和任务目标下具有接近的演化机制与决策意义。 CastEmbedding 的目标是让 embedding 空间中的近邻样本在数值模式、情境语义、未来趋势和任务价值上同时相近。

核心问题

CastEmbedding 关注的不是“如何把时间序列压缩成向量”这一单点问题，而是如何构建一个面向时序检索、推理和预测的统一语义空间。

01

仅靠曲线形态相似会产生错误近邻

两个负荷曲线可能形态相似，但一个处于极端高温，另一个处于普通工作日，其未来走势和决策价值可能完全不同。

02

情境会改变未来演化路径

天气、节假日、政策、突发事件、区域状态和设备工况会改变未来趋势，embedding 需要显式建模这些条件。

03

相似性必须服务下游任务

用于预测迁移、样本级 RAG、记忆增强预测或智能体决策时，近邻应在任务目标和决策意义上可复用。

核心亮点

CastEmbedding 借鉴 M3-Embedding 的统一嵌入基础设施思想，但面向时间序列建立多情境、多功能、多粒度的表示学习框架。

C1

Context-conditioned Temporal Semantics

情境条件化时序语义

embedding 空间不只表达曲线形态，而是把历史观测、领域背景、事件信息、任务目标和预测 horizon 共同纳入表示。 近邻样本应在“给定情境下的未来意义”上相似。

C2

Future Evolution Similarity

未来演化相似性

模型显式学习未来趋势、变化方向、变化点和演化机制上的一致性。 当两个历史窗口形态相似但未来路径不同，future-aware head 应降低它们的近邻可信度。

C3

Task-oriented Similarity

任务导向相似性

不同任务需要不同近邻：预测迁移关注未来误差改善，异常检测关注异常机制，智能体推理关注可复用证据。 CastEmbedding 将任务目标纳入相似性学习。

C4

Multi-granularity Temporal Retrieval

多粒度时序检索

表示空间同时支持全局序列检索、局部 patch 动态检索、样本级检索和任务级检索， 让同一套 embedding 能服务分类、聚类、RAG 和预测智能体。

C5

Temporal Self-Knowledge Distillation

时序自知识蒸馏

多个相似性 head 互相校正：pattern head、context head、future head 和 task head 形成更强 teacher signal， 避免模型退化为单纯曲线匹配或浅层文本对齐。

技术体系

CastEmbedding 由 temporal encoder、context encoder、context-conditioned fusion module、多视角相似性 heads 和自知识蒸馏机制组成。

多视角相似性矩阵

相似性视角回答的问题主要服务场景

Contextual Semantic Similarity 哪些样本处于相似领域、事件和外部条件中？ 情境检索、跨域迁移、证据召回

Task-oriented Similarity 哪些样本对当前任务具有相似决策价值？ Agentic TSF、记忆增强预测、任务级检索

训练目标

CastEmbedding 的训练目标不是单一 contrastive objective，而是将多种监督信号组织为一个统一的时序语义学习过程。

Objective 1

Context-aware Contrastive Learning

正负样本不只根据曲线形态构造，还要考虑领域、事件、预测 horizon 和任务目标。 形态相似但情境冲突的样本应被区分。

Context-conditioned Positive / Negative Pairs
Objective 2

Future-aware Supervision

引入未来趋势、变化方向、变化点和预测误差改善作为监督信号， 让 embedding 直接服务预测和未来演化检索。

Future Trajectory · Direction · Turning Point
Objective 3

Temporal Self-Knowledge Distillation

融合多个 similarity head 的判断形成 teacher signal，再蒸馏回各个分支， 使 pattern、context、future 和 task 视角相互增强。

Teacher Signal · Multi-head Correction · Unified Space

应用场景

CastEmbedding 的目标是成为时间序列检索、记忆、预测和智能体推理的基础组件。

样本级时序检索

检索历史相似样本、相似局部片段、相似事件背景和相似未来演化路径。

Time Series RAG

为预测模型或 LLM 提供近邻案例、上下文证据、失败经验和历史分析结论。

记忆增强预测

在 Memory 环境中召回高价值历史轨迹，辅助预测修正和不确定性分析。

预测迁移与 Few-shot 适配

根据情境和未来演化语义寻找可迁移样本，降低新领域标注和训练成本。

异常与风险案例检索

检索相似异常机制和历史风险案例，用于诊断、预警和解释生成。

Agentic Time Series Forecasting

作为智能体的检索和记忆基础，让工具调用与推理过程获得更可靠证据。

评估体系

CastEmbedding 的评估应同时覆盖 embedding 空间质量、近邻语义有效性和下游任务增益。

Retrieval Quality

检索质量

评估 Recall@K、nDCG、MRR、样本级检索命中率和局部片段检索质量。

Context Consistency

情境一致性

验证近邻样本是否具有一致领域背景、事件条件、外生变量影响和任务目标。

Future Validity

未来演化有效性

衡量近邻样本在未来趋势方向、变化点、预测误差和风险演化上的一致性。

Downstream Gain

下游任务增益

评估 embedding 对预测、异常检测、RAG、记忆增强预测和智能体推理的实际提升。

时间序列 embedding 不应只回答“哪些历史曲线相似”，而应回答 “哪些样本在给定情境下具有相似的未来演化意义”。

Existing time series embeddings mainly learn pattern similarity, while CastEmbedding aims to learn context-conditioned temporal semantics, where nearby samples are similar not only in curve shape, but also in contextual meaning, future evolution, and task-oriented decision value.
